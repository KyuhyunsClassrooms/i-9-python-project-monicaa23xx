# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:수술 후 환자 통증(NRS) 및 무통주사(PCA) 부작용 실시간 모니터링 시스템

이 프로그램을 만든 이유:미래에 간호사가 되어 환자 기록과 수치를 효율적으로 관리하여 행정 부담을 줄이고 싶기 때문입니다. 수술 후 환자가 무통주사를 맞을 때 발생하는 통증 점수와 부작용(오심, 어지러움 등)을 간호사가 일일이 수기로 확인하고 판단하는 번거로움을 줄이고, 프로그램으로 조치가 필요한 위험 환자를 실시간으로 파악하여 신속하고 정확하게 환자를 돌보기 위해 이 프로그램을 만들었습니다.

## 2. 문제 분해 및 모델링

### 입력
초기 데이터: 기존에 등록된 병동 환자 3명의 정보 (환자 번호, 수술명, 통증 점수, 부작용 증상)

사용자 입력: 신규 환자의 번호(num), 수술명(op), 통증 점수(pain), 부작용 증상(side)을 키보드(input())로 입력받음

### 처리
- 2차원 리스트 탐색: for 반복문을 사용하여 병동 환자 리스트의 행을 하나씩 추출

복합 조건 판정 알고리즘:

1순위: 환자의 부작용 열(patient[3])이 '없음'이 아니라면 ➡️ [주입 중단 및 부작용 완화 약물 투여 후 재주입] 판단

2순위: 부작용은 없으나 통증 점수 열(patient[2])이 4점 이상이면 ➡️ [추가 주입] 판단

3순위: 그 외의 경우 ➡️ [유지 및 안정] 판단

1) 함수 분해를 통한 역할별 처리 구조
[출력 및 반복 처리] monitor_all_patients(): 프로그램 시작 시 실행되며, 전체 환자 데이터베이스인 2차원 리스트 patients를 for 반복문으로 순회합니다. 가로줄(행) 데이터를 한 명씩 순서대로 추출하여 진단 함수로 전달하는 흐름을 제어합니다.

[진단 및 조건 처리] analyze_patient(patient): 반복문 내에서 추출된 개별 환자의 데이터를 매개변수로 전달받아, 세부 간호 항목(열 데이터)을 기반으로 환자의 안전 상태를 판정하고 맞춤형 지시서 문장(return)을 생성합니다.

[입력 및 누적 처리] register_new_patient(): 키보드(input())를 통해 신규 환자의 정보를 입력받아 정수형 변환(int()) 등 데이터 정제를 거친 후, 1차원 리스트로 묶어 기존 2차원 리스트에 append()로 누적 저장 및 즉시 진단 처리를 수행합니다.

2) 알고리즘 판단 기준에 따른 다중 분기 처리 구조 (경우의 수 분류)
analyze_patient 함수는 환자의 생체 안전을 최우선으로 확보하기 위해 다음과 같이 3가지 경우의 수로 단계를 나누어 복합 조건을 처리합니다.

[경우 1] 부작용이 발생한 경우 (최우선 순위 처리)

판단 조건: 환자의 부작용 열 데이터가 '없음'이 아닐 때 (if patient[3] != "없음")

처리 내용: 통증 점수의 높고 낮음과 관계없이 환자의 안전을 위협하는 약물 부작용(오심, 어지러움 등)이 발견되었으므로, 최우선 간호 원칙에 따라 즉시 [주입 중단 및 부작용 완화 약물 투여 후 재주입] 지시서 문자열을 반환합니다.

[경우 2] 부작용은 없으나 통증이 심한 경우 (2순위 처리)

판단 조건: 부작용은 '없음'이지만, 통증 점수 열 데이터가 4점 이상일 때 (elif patient[2] >= 4)

처리 내용: 부작용은 없어 안전하지만 환자가 조절되지 않는 중등도 이상의 통증을 겪고 있으므로, 무통주사 추가 버튼(Bolus)을 누르거나 추가적인 진통제 처방을 고려하도록 [추가 주입] 지시서 문자열을 반환합니다.

[경우 3] 통증과 부작용이 모두 안정적인 경우 (3순위 처리)

판단 조건: 위의 두 가지 위험 상황에 모두 해당하지 않을 때 (else)

처리 내용: 통증 점수가 4점 미만으로 잘 조절되고 있으며 부작용도 없는 완벽한 안녕 상태이므로, 현재 약물 주입 속도를 그대로 유지하도록 [유지] 지시서 문자열을 반환합니다.

데이터 누적: 신규 환자 정보를 1차원 리스트로 묶어 기존 2차원 리스트에 append()로 추가

통증도 4 이상이고 부작용도 있을 때 부작용 먼저 판단하는 것은 간호학적으로 무통주사 중 심한 부작용(오심, 구토, 호흡 이상 등)이 나타날 시 환자의 안전에 즉각적인 위협이 될 수 있기 때문이다. 통증이 조금 있더라도 심하게 부작용이 발견되면 주입을 즉시 중단하거나 부작용 완화 약물을 투여한 후 재주입하는 것이 최우선 간호 원칙이다.
### 출력
현재 병동에 있는 환자들의 실시간 상태 분석 결과 및 맞춤형 간호 지시서 출력

신규 환자가 추가된 후, 해당 환자의 정보 반영 결과와 간호 지시서 출력

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:patients

한 행의 의미:병동에 입원하여 무통주사(PCA)를 맞고 있는 환자 1명의 전체 정보

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |환자번호  |P001  |
| 1번 열 |환자가 받은 수술 종류  |복강경 수술  |
| 2번 열 |환자가 느끼는 통증 점수(0~10)  |6  |
| 3번 열 |무통주사로 인한 심한 부작용 증상  | 오심 |

이 2차원 리스트를 사용한 이유:
환자 여러 명의 데이터(행)를 관리하면서, 각 환자가 가진 여러 종류의 세부 간호 항목 데이터(열)를 표 형태로 묶어서 한 번에 체계적으로 관리하고 다루기 위해 사용했습니다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| analyze_patient | 환자 1명의 상태를 진단하여 알맞은 간호 지시 문장을 생성함 | patient |분석 결과 메시지(return 문자열)  |
|monitor_all_patients  |병동에 있는 전체 환자를 순서대로 확인하며 실시간 상태 분석 결과를 출력함  |없음  | 전체 환자 분석 현황(화면 출력) |
| register_new_patient | 키보드로 새 환자의 정보를 입력받아 기존 환자 데이터 리스트에 반영함 |없음  | 등록 완료 및 새 환자 분석 결과(화면 출력) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:analyze_patient

설명:환자 1명의 데이터를 리스트 형태로 전달받아 환자의 통증과 부작용 상태를 비교·진단하는 알고리즘 함수입니다. 임상 간호 원칙에 입각하여, 환자의 생체 안전을 위협할 수 있는 부작용 열(patient[3])이 '없음'이 아닌지 1순위로 검사하여 즉시 주입 중단 및 완화 약물 투여를 지시합니다. 부작용이 없을 때에만 2순위로 통증 점수 열(patient[2])이 4점 이상인지 비교하여 추가 주입을 결정하고, 모든 상태가 안정적이면 유지를 지시하는 문자열을 반환(return)합니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
analyze_patient 함수 내부에서 if patient[3] != "없음"을 검사하여 환자에게 무통주사 부작용 증상이 생겼는지 판별하였고, elif patient[2] >= 4 조건을 통해 환자가 조절되지 않는 중등도 이상의 심한 통증을 겪고 있는지 검사하였습니다.

### 반복문 사용 부분

무엇을 반복 처리했는가?
analyze_patient 함수 내부에서 if patient[3] != "없음"을 검사하여 환자에게 무통주사 부작용 증상이 생겼는지 판별하였고, elif patient[2] >= 4 조건을 통해 환자가 조절되지 않는 중등도 이상의 심한 통증을 겪고 있는지 검사하였습니다.

### 조건문과 반복문이 함께 작동한 부분

monitor_all_patients 함수가 실행되면 for 반복문이 돌아가면서 병동에 입원한 환자의 가로줄 데이터(p)를 한 명씩 순서대로 꺼냅니다. 그리고 반복문 내부에서 조건문 구조를 가진 analyze_patient(p) 함수를 호출하여 데이터를 넘겨줍니다. 이로 인해 단 한 명의 환자도 누락하지 않고 전체 환자의 통증과 부작용을 컴퓨터가 실시간으로 분석해 내어 알맞은 간호 지시서를 화면에 띄울 수 있게 함께 작동합니다.
