# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

수술 후 환자 통증(NRS) 기록 및 무통주사(PCA) 부작용 모니터링기

## 2. 문제 상황



내 설명: 수술 후 통증 지수를 기록하고 무통주사 투입한 상태에서 부작용 여부를 분석해 무통주사 투입을 중단할 지 유지할 지 한눈에 제공해주는 프로그램


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 키보드(input())를 통한 신규 입원 환자 정보: 환자 번호, 수술명, 통증 점수(0~10 정수형), 부작용 증상 문자열

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 2차원 리스트 탐색: for 반복문을 사용하여 병동 환자 리스트의 행을 하나씩 추출

복합 조건 판정 알고리즘:

1순위: 환자의 부작용 열(patient[3])이 '없음'이 아니라면 ➡️ [주입 중단] 판단

2순위: 부작용은 없으나 통증 점수 열(patient[2])이 4점 이상이면 ➡️ [추가 주입] 판단

3순위: 그 외의 경우 ➡️ [유지 및 안정] 판단

데이터 누적: 신규 환자 정보를 1차원 리스트로 묶어 기존 2차원 리스트에 append()로 추가

통증도 4 이상이고 부작용도 있을 때 부작용 먼저 판단하여 주입을 중단하는 것은 의학적으로 무통주사의 부작용 발생 시 위험해질 가능성이 더 크기에 이를 고려하여 통증도보다 부작용을 우선시한 것이다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 병동 내 전체 환자의 실시간 진단 및 간호 지시서 결과 출력

신규 등록된 환자의 정보 반영 결과 및 맞춤형 간호 지시서 출력

---

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:
patients

```python
patients = [
    ["P001", "복강경수술", 6, "오심"],
    ["P002", "충수절제술", 2, "없음"],
    ["P003", "제왕절개술", 3, "어지러움"]
]
```

### 한 행의 의미
행(Row): 입원한 환자 1명의 독립된 정보 가로줄 (가상 환자 데이터 활용)

### 각 열의 의미

열(Column): 환자의 세부 간호 데이터 세로줄 (총 4개 열)

[0] 열: 환자 번호 (예: "P001")

[1] 열: 수술명 (예: "복강경수술")

[2] 열: 통증 점수 (0~10 범위의 정수 데이터)

[3] 열: 부작용 증상 (예: "오심", "어지러움", "없음")

---

## 5. 함수 설계

함수 이름:analyze_patient
역할:환자의 통증 점수와 부작용 여부를 확인하여 진단 메시지를 결정한다.
입력값:patient (환자 1명의 정보가 담긴 리스트)
출력값 및 반환값:진단 결과 문자열 (🚨주입 중단 / ⚠️추가 주입 / ✅유지) 반환
---

## 6. 사용할 제어구조

### 조건문을 사용할 부분


내 계획:환자 데이터 중 부작용 항목이 '없음'이 아니면 즉시 주입을 중단하고, 부작용이 없더라도 통증 점수가 4점 이상이면 추가 주입을 고려하도록 분기하여 처리한다. 모두 해당하지 않는 안정적인 상태일 때는 유지를 선택한다.


### 반복문을 사용할 부분


내 계획:for 반복문을 사용하여 병동 환자 현황이 저장된 2차원 리스트(patients)에서 환자 데이터를 한 명씩 순서대로 꺼내어 실시간 상태 분석 함수로 전달하고 결과를 출력한다.

## 7. 예상되는 예외 상황


1. 신규 환자 등록 시 '통증 점수' 입력란에 0~10 사이의 숫자가 아닌 문자(예: "오점")나 음수를 입력하여 에러가 발생하거나 잘못된 데이터가 저장되는 상황
2. 부작용 증상을 입력할 때 '없음' 대신 실수로 공백(스페이스바)이나 아무것도 입력하지 않고 엔터를 쳐서 조건문 판정(patient[3] != "없음")에 혼선이 생기는 상황
